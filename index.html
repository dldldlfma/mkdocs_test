mkdocs github-deploy

readthedocs로 만든 내용 정리 사이트를 github에 올리는걸 실험해 보려고 작성하였습니다.

사용명령

  • mkdocs new [dir-name] - mkdocs 프로젝트를 만들때 사용합니다.
  • mkdocs serve - 현재 컴퓨터를 서버로 하여 mkdocs를 보여줍니다.
  • mkdocs build - documentation site를 build합니다..
  • mkdocs -h - help message를 보여줍니다.
  • mkdocs gh-deploy - github에 연결된 repo에 본 사이트를 업로드합니다.

github-deploy

github에 deploy를 하게되면 github.io 사이트를 통해서 자신이 작성한 mkdocs를 웹상에서 볼수 있게 해줍니다.

예를 들어서 현재 제 id가 dldldlfma 인데

repo 이름을 mkdocs_test로 했다면

mkdocs gh-deploy

위 명령이 정상 실행됬을때

현재 작성된 mkdocs를

https://dldldlfma.github.io/mkdocs_test

에서 볼 수 있다는 뜻입니다.

위 작업은 아래와 같은 순서로 진행됩니다.

  1. mkdocs 프로젝트를 만들고 작성한다.
mkdocs new my_prj

위와 같이 하면된다.

  1. github 에 repo를 만들고 github와 연동한다.

  2. 현재 만들어져 있는 mkdocs 프로젝트 폴더에서 git init를 친다.

  3. github에 repository를 만들고 mkdocs 프로젝트 폴더에서 git remote로 해당 repo를 연결한다.

  4. mkdocs gh-deploy 를 수행한다.

mkdocs gh-deploy

위 3단계의 과정을 거치면 작성한 repo를 github.io에서 확인할 수 있게 됩니다.
